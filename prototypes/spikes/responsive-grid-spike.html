Responsive-grid spike — CONSTITUTION.md §5

What this proves: the same 12-column grid used across all 25 style-tile prototypes stays fully responsive down to 320px with no horizontal page scroll, and defines the breakpoint (1024px) at which the article/sidebar 8/4 split collapses to a stacked full-width layout. None of the 25 prototypes tested anything narrower than their fixed desktop screenshot — this spike exists to close that gap.

How to test: resize the viewport down to 320px. Check for a horizontal scrollbar (there should be none). Above 1024px, article/sidebar sit side by side (8 cols / 4 cols). Below it, both stack full-width in source order — still a 12-column grid throughout, per §5 (the regions just each claim all 12 columns instead of 8+4).

Overflow stress-test content

Ordinary paragraph text wraps normally, no special handling needed for this. But here is a deliberately unbroken long token to test wrapping safety: https://example.com/a/very/long/path/segment/that/has/no/spaces/and/could/force/horizontal/overflow/if/not/handled/12345678901234567890

A long, hyphen-free chemical-style identifier as another stress case: 1,2,3,4,5,6,7,8-octahydronaphthalene-carboxylic-acid-derivative-compound-reference-code-XJ992817364

Code block (should scroll internally, not push the page wide)

Short sidebar content. At narrow widths this stacks below the article, still spanning the full 12 columns.

Checklist

  • ✅ No horizontal page scroll at 320px (verified via screenshot at that exact width).
  • ✅ Long unbroken tokens (URLs, identifiers) wrap via overflow-wrap: break-word rather than forcing page-level overflow.
  • ✅ Code blocks scroll internally (overflow-x: auto on <pre>, already used in all 25 prototypes) rather than widening the page.
  • ✅ Still a genuine 12-column grid at every width — narrow layouts just span all 12 columns instead of 8+4, not a different column count.

Breakpoint decided by this spike: 1024px for the article/sidebar stack point. This is a shared, group-agnostic mechanism value — each style's own docs/design-tokens-<group>.md can now cite this instead of leaving it TBD.
